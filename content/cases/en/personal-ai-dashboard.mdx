## What problem it solves

Most AI assistants give advice. This one acts — it reads your real data and can schedule your day, mark tasks complete, update your learning plan, and set working hours, all from a single message in natural language.

## Project essence

A personal productivity system built for one user (myself) with 7 learning tracks, a streak system, XP gamification, and an AI coach persona backed by Claude claude-sonnet-4-6. The AI has real-time access to your schedule, tasks, journal entries, and XP — and can execute 7 types of data mutations directly from the chat window.

Built on Next.js 14 App Router with TypeScript, Tailwind CSS, shadcn/ui, and Zustand with localStorage persistence.

## How it works

- **Structured AI agent** — every Claude response returns two tagged blocks: `<RESPONSE>` for the text shown to the user, and `<ACTIONS>[{...}]</ACTIONS>` with a JSON array of operations. The frontend parses the JSON and executes them immediately — no server required
- **7 action types** — `addTask`, `completeTask`, `uncompleteTask`, `skipTask`, `updateSchedule`, `setDayJobs`, `resetData`. Claude decides which to call based on what the user said in natural language
- **Prompt caching** — the 700+ token static system prompt (persona, rules, action spec) is cached with Anthropic's `prompt-caching-2024-07-31` beta using `cache_control: ephemeral`. After the first call, subsequent requests cost 90% less for that portion
- **Dynamic context injection** — each request includes fresh real-time state: today's tasks with IDs, current streak, XP per track, upcoming scheduled tasks, and the last 7 days of journal entries. Claude makes decisions with full situational awareness
- **7 learning tracks** — AI, Design, Self-development, Media buying, English, Polish, Gym — each with XP, colour coding, and streak tracking
- **Career module** — vacancy pipeline (`saved → applied → replied → interview → offer → rejected`), CV templates per role, notes and lessons learned per application
- **Journal with AI analysis** — daily entries with monthly coach profiles: Claude summarises patterns across the month into a persistent "profile" that gets injected alongside fresh entries in future sessions

## Challenges in the process

**Reliable structured output.** Getting Claude to always return valid JSON inside `<ACTIONS>` tags required careful prompt engineering — especially when the response was long. The solution: explicit format rules at the top of the prompt with a worked example, and a clear instruction that `<ACTIONS>[]</ACTIONS>` is required even when there are no actions.

**Prompt caching boundaries.** Caching requires hard separation between static and dynamic content — any change to the static portion invalidates the cache. The system prompt is split into two blocks in code: `STATIC_PROMPT` (cached) and `dynamicPrompt` (rebuilt every request). Comments in the code make this boundary explicit so it can't accidentally be broken.

**Streak edge cases.** Streak logic has more edge cases than expected: freeze mechanics (one allowed per calendar month), timezone handling, the difference between a skipped day vs. a missed day, and what counts as an "active" day for the streak counter. Each required explicit logic, not assumptions.

## Limitations

- **Personal project, not productized** — designed around one person's specific workflow. The AI persona, tracks, and action spec are all hardcoded for this use case
- **Client-side state only** — everything lives in localStorage via Zustand persist. No backend database. If browser storage is cleared, all data is lost
- **History window cap** — only the last 40 messages are sent to the API to avoid token overflow on very long conversations. Older messages remain visible in the UI but are dropped from the API call

## Financial perspective

Without caching: ~$0.003 per message (claude-sonnet-4-6 input pricing at full token count). With `cache_control: ephemeral` on the 700-token static prompt: ~$0.0003 for that portion on every call after the first.

At personal usage scale (~200 messages/month) the saving is small. The value of this implementation is the **pattern** — the same architecture applied to a production SaaS with 10,000+ daily messages saves hundreds of dollars per month with zero change to the user experience.

This is what cost-aware AI engineering looks like: not cheaper models, but smarter use of the API you already have.
